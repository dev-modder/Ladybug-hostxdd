## рџ“¦ Panel Bots (Custom Bot Upload)

The Panel Bots feature allows developers to upload and run their own custom bots!

### Uploading a Bot
1. Click **"рџ“¦ Panel Bots"** in the header or navigate to `/panel-bots.html`
2. Click **"рџ“¤ Upload Bot"** or **"рџ”— Upload from GitHub"**
3. Fill in:
   - **Bot Name**: A friendly name for your bot
   - **Description**: What your bot does
   - **Entry Point**: The main JavaScript file (default: `index.js`)
   - **Bot ZIP File**: A ZIP file containing your bot code OR
   - **GitHub URL**: Clone directly from a GitHub repository
4. Click **"Upload Bot"**

### ZIP File Requirements
- Must contain a valid Node.js project
- Should include `package.json` if dependencies are needed
- The entry point file must exist (default: `index.js`)
- Dependencies will be auto-installed on upload

### GitHub Upload
- Provide the repository URL (e.g., `https://github.com/user/repo`)
- Specify the branch (default: `main`)
- The repository will be cloned and dependencies installed automatically

### Running Your Bot
1. Go back to the main Dashboard
2. Click **"Add New Session"**
3. Fill in the session details
4. Select your uploaded bot from the **"Panel Bot"** dropdown
5. Save and start your session!

### Bot Environment Variables
When running a panel bot, these environment variables are available:
- `SESSION_ID`: The session ID string
- `BOT_NAME`: The bot name
- `PREFIX`: The command prefix
- `OWNER_NUMBER`: The owner's WhatsApp number
- `TZ`: The timezone
- `BOT_ID`: The panel bot ID

### File Browser
- View and edit bot files directly from the dashboard
- Navigate directories and view file contents
- Edit `.js`, `.json`, `.env`, and other text files

- `DELETE /api/panel-bots/:id` - Delete bot

## рџ—„пёЏ MongoDB Integration (NEW in V5.1)

LADYBUGNODES now supports MongoDB for scalable, persistent data storage!

### Features

- **Automatic Fallback**: If MongoDB is not configured, the system automatically uses file-based storage
- **Hybrid Data Layer**: Seamless switching between MongoDB and file storage
- **Enhanced API**: New `/api/v2/*` endpoints with better validation
- **Real-time Analytics**: Admin dashboard with live statistics
- **Transaction Tracking**: Complete coin transaction history

### Setting Up MongoDB

1. **Create a MongoDB Atlas Account** (Free Tier Available)
   - Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
   - Create a free cluster
   - Create a database user
   - Whitelist your IP (or use 0.0.0.0/0 for Render)

2. **Get Your Connection String**
   - Click "Connect" on your cluster
   - Choose "Connect your application"
   - Copy the connection string

3. **Add to Environment Variables**
   ```
   MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/ladybugnodes?retryWrites=true&w=majority
   ```

4. **Seed Initial Data** (Optional)
   ```bash
   npm run seed
   ```

### MongoDB Status Endpoint

Check MongoDB connection status:
```
GET /api/mongodb/status
```

Response:
```json
{
  "connected": true,
```
